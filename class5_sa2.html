Conditionals & Decision Making

A Conditional is a way to tell what might happen if something else happens first. It helps computers and people make decisions.

IF: It is raining
THEN: Take an umbrella

Why Conditionals Are Important

✔ Help us make smart choices
✔ Help solve problems
✔ Keep us safe (Example: Stop at red light)

Data & Information

Data is a collection of facts like numbers, words, or images.
Information is organized data that makes sense.

Data: 85, 90, 78
Information: Class average marks

Why Organize Data?

✔ Easy to find information
✔ Saves time
✔ Helps in decision-making

Data Structures

A Data Structure is a way to organize and store data.

List: Ordered items
Table: Rows & Columns
Stack: LIFO (Last In First Out)
Queue: FIFO (First In First Out)

Real-Life Examples

✔ Undo Button → Stack
✔ Printer Line → Queue
✔ Marksheet → Table

Charts & Graphs

Charts help us understand data visually.

Bar Graph: Compare items
Line Graph: Show change over time
Pie Chart: Show parts of a whole
Pictograph: Uses pictures

Where We See Charts

✔ News reports
✔ Weather forecasts
✔ School results

Functions & Variables

A Function is a named set of instructions.
A Variable is a container that stores changing values.

Input → Data entered
Process → Program runs
Output → Final result

Why Use Functions?

✔ Organize code
✔ Reuse instructions
✔ Make programs easier to read

Internet & Debugging

The Internet is a global network connecting computers.

1. REQUEST: Type website address
2. ROUTER: Finds path
3. SERVER: Sends website data
4. DISPLAY: Shows on screen

Debugging

✔ Bug = Coding error
✔ Debugging = Finding and fixing errors
✔ Test step-by-step to solve problems

Bugs & Debugging

A Bug is a mistake in code. Debugging is the detective process of finding and fixing that mistake to solve the problem.

Digital Art Tools
